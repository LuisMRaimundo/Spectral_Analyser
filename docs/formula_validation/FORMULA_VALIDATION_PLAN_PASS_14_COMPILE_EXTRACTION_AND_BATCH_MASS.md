# Formula Validation Plan — Pass 14 — Compile Extraction and Batch Mass

## 1. Scope

This plan defines small, hand-checkable numerical fixtures for formulas documented in `docs/formula_extraction/FORMULA_EXTRACTION_TABLE_PASS_14_COMPILE_EXTRACTION_AND_BATCH_MASS.md` (Pass 14): **`compile_metrics.py`** and **`finalize_batch_power_mass_summary`** in `audio_analysis/super_audio_analyzer.py`. It is intended for later `pytest` implementation; **no tests are created here**. Prefer **direct function calls**; use **tiny temporary `.xlsx`** built with `pandas.ExcelWriter` only where extraction requires a workbook. **No** full GUI / corpus / integrated Excel workflows.

## 3. Deferred / human-review cases

Do **not** treat these as settled automated gold requirements:

- Scientific optimality of **`get_frequency_dependent_alpha`** register table.
- Whether **`apply_frequency_dependent_normalization`** should use the **integral proxy** \(E(N)\) vs a **discrete** \(\sum n^{-\alpha}\) sum.
- Design validity of **`"pdf"`** vs **`"current"`** weighted-index schemes and their weights.
- Design validity of **substring-based** `"Dissonance"` column harvesting in `extract_dissonance_metrics`.
- **Full Excel workbook** integration, GUI export parity, and **openpyxl** layout.
- Internals of **`data_integrity.robust_normalize`** (percentile/IQR/z-score definitions).
